# 構成選定ガイド（Windmill自作 vs Hermes Agent）

## 結論（推奨）

- **短期**: Hermes Agentで1週間PoC
- **並行**: Windmill構成ドキュメントを維持
- **最終判断**: PoC受け入れ基準を満たせばHermes採用、満たさなければWindmill本命

## 比較

| 観点 | Windmillで構成 | Hermes Agentで構成 |
|---|---|---|
| 立ち上がり速度 | △ | ◎ |
| 構成の透明性 | ◎ | ○ |
| 障害切り分け | ◎ | △ |
| Discord連携の初速 | △ | ◎ |
| Memory/Cron/MCPの初速 | △（段階実装） | ◎（統合機能） |
| 長期運用での制御性 | ◎ | ○ |

## 選定ルール

- 次を最優先するなら **Windmill**
  - 制御性
  - 可観測性
  - 監査/承認の厳密運用

- 次を最優先するなら **Hermes**
  - 立ち上がりの速さ
  - 既成機能の豊富さ
  - 早期の利用体験検証

## 補足

このリポジトリでは、両案の実行順を次のように固定する。

1. HermesでPoC（1週間）
2. 受け入れ基準レビュー
3. 本採用を決定（Hermes継続 or Windmill実装本格化）
